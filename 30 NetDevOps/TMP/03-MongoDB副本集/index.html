    03 MongoDB副本集

    MongoDB 副本集¶

    我们建议部署 MongoDB 副本集。

    MongoDB 不必在专用服务器上运行 Graylog 生成的工作负载，但您应该遵循 MongoDB 文档中有关架构的建议。最重要的是副本集中有奇数个 MongoDB 服务器。

    在大多数设置中，每个 Graylog 服务器还将托管一个 MongoDB 实例，该实例是同一副本集的一部分，并与集群中的所有其他节点共享数据。

    **提示：**为了避免未经授权访问您的 MongoDB 数据库，MongoDB 副本集应设置身份验证。

    正确的工作步骤顺序应如下：

    1. 创建副本集 ( rs01)
    2. 创建数据库（graylog）
    3. 创建用于访问数据库的用户帐户，该帐户具有角色readWrite和dbAdmin。

    bind_ip如果您的 MongoDB 需要通过网络进行访问，您应该在配置中设置 IP 。
