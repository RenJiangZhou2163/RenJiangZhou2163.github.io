    命名空间¶

    在Kubernetes中，命名空间（Namespace）用于将集群中的资源进行逻辑上的隔离。查找和管理命名空间可以通过以下几种方法实现：

    1. 列出所有命名空间¶

    使用 kubectl get namespaces 命令可以列出集群中的所有命名空间：

    kubectl get namespaces

    输出示例：

    NAME              STATUS   AGE
default           Active   10d
kube-system       Active   10d
kube-public       Active   10d
my-namespace      Active   5d

    2. 获取详细信息¶

    如果你想查看某个特定命名空间的详细信息，可以使用 kubectl describe namespace 命令：

    kubectl describe namespace <namespace-name>

    例如：

    kubectl describe namespace my-namespace

    这将显示该命名空间的详细信息，包括标签、注释和状态等。

    3. 使用 kubectl get all 查看资源¶

    你还可以使用 kubectl get all 命令来查看特定命名空间中的所有资源。例如，查看 default 命名空间中的所有资源：

    kubectl get all -n default

    输出示例：

    NAME                          READY   STATUS    RESTARTS   AGE
pod/my-pod-1234567890-abcde   1/1     Running   0          5m

NAME                 TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)          AGE
service/my-service   ClusterIP   10.96.0.1       <none>        80/TCP           10d

NAME                      READY   UP-TO-DATE   AVAILABLE   AGE
deployment.apps/my-app    1/1     1            1           5m

NAME                                 DESIRED   CURRENT   READY   AGE
replicaset.apps/my-pod-1234567890    1         1         1       5m

    4. 使用 kubens 工具（可选）¶

    如果你频繁切换命名空间，可以考虑使用 kubens 工具，它是一个方便的命令行工具，用于快速切换当前上下文的命名空间。

    安装 kubens¶

    你可以从GitHub仓库下载并安装kubens：

    # 下载最新版本的 kubens (假设为 Linux 平台)
curl -Lo kubens https://github.com/ahmetb/kubectx/releases/download/v0.9.3/kubens_v0.9.3_linux_x86_64

# 为 kubens 添加可执行权限并移动到 /usr/local/bin/
chmod +x kubens
sudo mv kubens /usr/local/bin/

    使用 kubens¶

    列出所有命名空间：

    kubens

    切换到特定命名空间：

    kubens <namespace-name>

    例如，切换到 my-namespace 命名空间：

    kubens my-namespace

    以上方法都可以帮助你查找和管理Kubernetes集群中的命名空间。根据具体需求选择最适合的方法进行操作。

    如何查看k8s集群IP？¶

    在Kubernetes集群中查看节点（Node）开放的端口可以通过以下几种方法实现：

    1. 使用 kubectl describe node¶

    使用 kubectl describe node 命令可以查看节点的详细信息，包括节点上运行的Pod和相关端口。

    kubectl describe node <node-name>

    例如：

    kubectl describe node my-node

    在输出中，你可以找到与网络和端口相关的信息，不过这种方法可能不会直接显示所有开放的端口。

    3. 使用 netstat 或 ss 命令¶

    你也可以直接登录到Kubernetes节点上，使用 netstat 或 ss 命令查看开放的端口。

    使用 netstat 命令：¶

    sudo netstat -tuln

    使用 ss 命令：¶

    sudo ss -tuln

    这些命令会列出所有正在监听的端口及其对应的服务。

    4. 使用 nmap 工具¶

    如果你有权限，可以使用网络扫描工具如 nmap 来扫描节点开放的端口。首先需要安装 nmap，然后运行以下命令：

    nmap <node-ip>

    例如：

    nmap 192.168.1.10

    这将扫描指定节点IP上所有开放的端口。

    5. 查看防火墙规则¶

    如果你的集群启用了防火墙（如iptables或firewalld），你可以查看防火墙规则来了解哪些端口被允许通过。

    对于iptables:¶

    sudo iptables -L -n -v

    对于firewalld:¶

    sudo firewall-cmd --list-all-zones

    以上几种方法都可以帮助你查看Kubernetes节点上开放的端口。根据具体需求选择最适合的方法进行操作。

    命令示例¶

    查看节点上的Service¶

    如果你想查看哪些服务暴露了特定的端口，可以使用以下命令列出所有Service，并检查它们的NodePort配置：

    kubectl get services -o wide

    示例

    [root@beilonghu-jk32g-01 ~]# kubectl get services -o wide | grep 28668
bk-gse-cluster                               ClusterIP   10.98.168.104    <none>        27707/TCP,28808/TCP,28668/TCP,48533/TCP,29405/TCP                                 6d16h   app=gse-cluster
bk-gse-cluster-headless                      ClusterIP   None             <none>        27707/TCP,28808/TCP,28668/TCP,48533/TCP,29405/TCP

[root@beilonghu-jk32g-01 ~]# kubectl get services -o wide | grep 28625
bk-gse-data                                  ClusterIP   10.96.72.170     <none>        59702/TCP,28625/TCP,58625/TCP,28626/UDP,29402/TCP                                 6d16h   app=gse-data
bk-gse-data-headless                         ClusterIP   None             <none>        59702/TCP,28625/TCP,58625/TCP,28626/UDP,29402/TCP                                 6d16h   app=gse-data

    然后使用 kubectl describe service 命令查看具体Service的详细信息：

    kubectl describe service <service-name> -n <namespace>

    例如：

    kubectl describe service my-service -n my-namespace

    在输出中，你可以找到类似以下的部分，显示服务暴露的端口信息：

    Type: NodePort
Port: 80/TCP
TargetPort: 8080/TCP
NodePort: 30080/TCP
